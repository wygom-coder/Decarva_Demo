# 클로드(Claude)에게 전달할 코드 검토 프롬프트

> **사용 방법:** 
> 아래의 `[프롬프트 시작]` 부터 `[프롬프트 끝]` 까지의 내용을 전체 복사하신 후, 기존에 띄워두신 클로드(Claude 3.5 Sonnet 등) 대화창에 그대로 붙여넣기 하시면 됩니다. 
> 만약 깃허브 전체 코드를 압축해서 첨부(또는 프로젝트 폴더 제공)하실 수 있다면 클로드가 훨씬 더 정확하게 분석할 수 있습니다.

### [프롬프트 시작]

**[역할 부여]**
너는 15년 차 시니어 Full-stack 엔지니어이자 보안(SecOps) 전문가야. 지금부터 내가 제공하는 B2B 조선/해운 기자재 중고 거래 플랫폼인 '해마 마켓(Haema Market)'의 프론트엔드 및 백엔드(Supabase) 구조를 심층적으로 리뷰하고 피드백을 줘.

**[프로젝트 기술 스택 및 현재 상태]**
1. **프론트엔드:** 순수 바닐라 JavaScript (ES6+), HTML5, CSS3 
2. **백엔드/DB:** Supabase (PostgreSQL, Auth, Storage, Realtime)
3. **최근 업데이트 완료된 마일스톤 (MVP V1.0):**
   - 카카오 소셜 로그인을 포함한 인증(Auth) UI 및 기업 정보(직함/회사명) 수집 로직 개편
   - `app_metadata.role = 'admin'` 기반의 관리자(Admin) 전용 백오피스 대시보드 구축 (매물/커뮤니티 글 강제 삭제 및 탈퇴 요청 큐 처리)
   - 오션 블루(Ocean Blue) & 바이브런트 오렌지(Vibrant Orange) 톤의 B2B 브랜드 컬러 전역 리팩토링 및 누끼 로고 교체
   - 총 8개 핵심 테이블(`products, posts, comments, chat_rooms, messages, quotes, likes, reviews`)에 대한 Supabase RLS(Row Level Security) 접근 제어 정책 완벽 적용

**[요청 사항: 심층 리뷰 및 최적화 진단]**
현재 MVP 수준의 기능 구현과 보안(RLS) 세팅은 모두 마친 상태야. 이 코드 및 아키텍처에 대하여 아래 4가지 관점에서 날카롭게 진단해 줘.

**1. 보안 결점 (Security Vulnerabilities) 체크**
- 작성된 RLS 정책(내가 첨부한 SQL 포함) 중에 해커가 우회(Bypass)할 수 있는 논리적 허점이나 엣지 케이스가 있는가?
- 프론트엔드 바닐라 JS 단에서 `innerHTML` 등을 사용해 발생할 수 있는 잠재적 XSS(크로스 사이트 스크립팅) 공격 취약점이 존재하는가? (현재 `escapeHtml` 헬퍼 함수를 제한적으로 사용 중임)
- 어드민 권한 체크(`app_metadata`)가 프론트엔드 UI를 넘어서 백엔드 쿼리에 안전하게 얽혀 있는가?

**2. 코드 최적화 및 구조 제안 (Code Optimization)**
- Vanilla JS가 비대해져서 모듈(`admin.js`, `mypage.js`, `product.js`, `chat.js` 등)로 파일을 나누어 놓았으나, 전역 변수나 상태 관리 측면에서 메모리 누수나 렌더링 병목 현상이 발생할 부분이 있는가?
- 비동기 처리(`async/await`) 및 Supabase API 호출 중 불필요하게 낭비되는 네트워크 트래픽(N+1 쿼리 등)이 있는가?

**3. 로직 및 예외 처리 (Logic Errors & Error Handling)**
- 회원 탈퇴 로직, 매물 삭제(Cascade), 혹은 경매 입찰 등의 로직에서 트랜잭션 에러 시 데이터가 꼬일 수 있는 부분이 있는가?
- 사용자가 네트워크가 끊기거나 이상한 데이터를 입력했을 때의 방어 로직이 충분한가?

**4. 향후 디벨롭(Next Step) 최우선 순위 제안**
- 이 플랫폼이 실제 오픈베타에 돌입하고 1,000명 이상의 기업 유저 트래픽을 받을 경우, 가장 먼저 무너질 수 있는 기술적 약점은 무엇이며 이를 보완하기 위해 다음에 어떤 기능을 우선적으로 개발해야 하는가?

**[진행 방식]**
전체 구조를 칭찬하기보다는, **가장 크리티컬하게 터질 수 있는 문제점 3가지**를 먼저 서론에 배치해 줘. 그리고 그에 대한 구체적인 개선 코드 스니펫이나 Supabase SQL 보완 코드를 제시해 주면 좋겠어.
